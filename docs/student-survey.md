# Student Survey

Use a 5-point Likert scale:

1. Strongly disagree
2. Disagree
3. Neutral
4. Agree
5. Strongly agree

## Required Items

| ID | Survey Item | Construct |
| --- | --- | --- |
| S1 | The progressive quality gate helped me identify newly introduced code issues earlier. | Perceived usefulness |
| S2 | Separating legacy lint debt from new lint violations reduced confusion during development. | Clarity |
| S3 | The quality gate added acceptable overhead to the development workflow. | Perceived effort |
| S4 | The MongoDB duplicate preflight check improved confidence before deployment. | Deployment confidence |
| S5 | The quality gate made the CI/CD process easier to understand. | Process understanding |
| S6 | The approach was suitable for a student project with existing technical debt. | Context fit |
| S7 | I would prefer using this progressive gate in future student web projects. | Adoption intention |

## Optional Open Questions

1. What part of the progressive quality gate was most useful?
2. What part caused the most friction?
3. What should be improved before using this approach in another course project?
